# Tài Liệu Vin-PipeLine

Toàn bộ tài liệu Markdown chính của pipeline được tập trung trong thư mục `docs/`.

## Mục lục

- [PIPELINE.md](./PIPELINE.md)
  Tài liệu kiến trúc, pipeline stages, stores, API, retrieval flow, và cấu hình runtime.

- [PRODUCTION_ANALYSIS.md](./PRODUCTION_ANALYSIS.md)
  Phân tích production risks, bottlenecks, failure chains, và ưu tiên xử lý.

- [overview.md](./overview.md)
  Góc nhìn tổng quan service trong toàn hệ thống và ranh giới trách nhiệm với BE.

- [onboarding.md](./onboarding.md)
  Hướng dẫn vào việc nhanh cho dev mới.

- [migration.md](./migration.md)
  Bối cảnh migrate từ pipeline cũ sang document pipeline hiện tại.

- [agent_prompt.md](./agent_prompt.md)
  Ghi chú triển khai và trạng thái làm việc phục vụ agent/dev flow.

- [de_onboard_design_mindset.md](./de_onboard_design_mindset.md)
  Design mindset và cách tư duy khi mở rộng service.

- [testing/TEST_LAYOUT.md](./testing/TEST_LAYOUT.md)
  Cấu trúc thư mục test hiện tại.

- [testing/TEST_EXPANSION_PLAN.md](./testing/TEST_EXPANSION_PLAN.md)
  Backlog mở rộng test coverage theo domain và production risks.

## Quy ước

- Tài liệu vận hành, phân tích, kiến trúc, onboarding, và test planning đều đặt trong `docs/`.
- Các file Markdown còn lại ngoài `docs/` chỉ nên là nội dung dữ liệu mẫu hoặc tài nguyên test fixture, không phải tài liệu chính.
