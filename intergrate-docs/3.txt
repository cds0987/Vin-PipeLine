🔐 Open Platform Authorization – Technical Summary
1. Overview
Hệ thống sử dụng OAuth 2.0 – Client Credentials Flow để xác thực giữa:

✅ Client Application (backend)
✅ Open Platform API

→ Không có user context → chỉ dùng cho server-to-server communication
Goal: Lấy access_token để gọi API.

2. High-level Flow
Plain TextClient (Backend)    |    | (1) POST /oauth2/token    |     client_id + client_secret    vAuth Server    |    | (2) return access_token (expires_in = 3600s)    vClient stores token    |    | (3) Call API with Authorization: Bearer token    vOpen Platform API    |    | (4) If expired → 401 → repeat step 1Show more lines

4. Production-grade Design
4.1 Token Manager Pattern
Thay vì gọi token mỗi request → dùng Token Manager Service
Example Architecture
Plain Text              +-------------------+              |   Token Manager   |              |-------------------|              | - cache token     |              | - refresh logic   |              +---------+---------+                        |                        vClient Services -----> API CallsShow more lines

4.2 Pseudocode
JavaScriptclass TokenManager {  constructor() {    this.token = null;    this.expiry = null;  }  async getToken() {    if (!this.token || Date.now() >= this.expiry) {      await this.refreshToken();    }    return this.token;  }  async refreshToken() {    const res = await axios.post('/oauth2/token', { ... });    this.token = res.data.access_token;    this.expiry = Date.now() + res.data.expires_in * 1000 - 5000;  }}Show more lines

5. Security Considerations (CRITICAL)
🔒 MUST DO

Store client_secret only on backend [Authorizat...tform Docs]
Use HTTPS only [Authorizat...tform Docs]
Mask token in logs
Use environment variables / secret manager

❌ NEVER

Commit secrets to Git
Send token to frontend
Hardcode credentials

7. Engineering Improvements
Nếu build production system → nên thêm:

✅ Retry with exponential backoff
✅ Circuit breaker
✅ Observability (log + metrics)
✅ Secret rotation
✅ Token pre-fetch (refresh trước expiry)


8. TL;DR (Senior-level)

Use OAuth2 Client Credentials
Cache token (don’t request per call)
Handle 401 → refresh → retry
Secure secrets aggressively
Centralize token logic (Token Manager)
